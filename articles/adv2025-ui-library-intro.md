## 概観

Material-UI や Chakra-UI などを初めとする UI ライブラリは多くの Web サービス・Web サイトで使用されています。これらのライブラリを使用することで、ボタンやセレクト、テーブルやダイアログなどの汎用的なコンポーネントを、自ら CSS やロジックを書くことなく使うことができるようになり、開発効率が大幅に上がります。
また、そういったコンポーネントの実装をライブラリ側で行い、統一させることができているため、まだフロントエンドエンジニアには十分に広まっているとは言い難い Web アクセシビリティ（以降 a11y と略す）の観点について、提供しているコンポーネントについては最低限のレベルを自動的に確保するという基盤的な役割も担っていると言えます。そこで今回のアドベントカレンダーでは、主要な UI ライブラリを調査しながら、いくつかの a11y のトピックについてどのように対応されているのかを見ていきます。具体的なトピックについては次のセクションで紹介します。

今回は去年のアドベントカレンダーのようにコンポーネント単位で見るようなことはあまりせず、a11y のトピックや WCAG の SC 単位で見ていくことになります。

## 書くことと書かないこと

### 書くこと

- a11y に関連する各トピックについて、UI ライブラリをいくつか見ながら比較・議論していきます
- 内部実装についても必要に応じて見ていきます
- a11y を語る上でほぼ必須の知識である WCAG の、Working Draft である WCAG 3.0 の話題についても見ていきます
- 参考リンクを頑張って多く貼るようにしたいと思っています
  - 内容の理解に特に役立つものは本文中に貼り、そうでないものは記事の最後にまとめて貼る予定です

### 書かないこと

- それぞれの UI ライブラリの、詳細な使い方についての説明などはしません
- 全体を通して特定の UI ライブラリを高く評価したり低く評価したりすることはしません
  - このトピックについてはこの UI ライブラリはまだ物足りない、という議論をすることはあるかもしれません
